## Prompt

sp.constitution

Create a constitution for Phase 1 of a Spec-Driven Todo Application.

Purpose:
Implement Phase 1 (Basic Task Management) of a Todo Application using Spec-Driven Development with Claude Code and Spec-Kit Plus.

Phase 1 Goal:
Deliver a console-based MVP that allows users to add, view, update, delete, and mark tasks as complete.

Core Principles:
- Spec-first development only
- No manual coding; all code generated by Claude Code
- Clear traceability from specs to implementation
- Clean separation of concerns (model, service, interface)
- Deterministic and predictable behavior

Constraints:
- Python 3.13+
- Console-based interface only
- In-memory data storage
- No database or file persistence
- No external libraries
- No web UI
- No advanced or intelligent features

Quality Standards:
- Readable and maintainable code
- Graceful handling of invalid inputs
- User-friendly CLI messages
- No feature beyond Phase 1 scope

Non-Goals:
- No priorities, tags, search, or filters
- No due dates or reminders
- No recurring tasks
- No natural language commands

Success Criteria:
- All Phase-1 acceptance scenarios pass
- User can fully manage tasks from the terminal
- Application is demo-ready for Hackathon II
